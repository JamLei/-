# Docker ElasticSearch

docker hub 的 https://hub.docker.com/_/elasticsearchopen in new window 。

当前（2021-04-02）的 7 版本最新的是 7.12.0 ，6 版本最新的是 6.8.15。（没有 latest 版本）

WARNING

支持 Window 7 操作系统的最后一个 es 版本是 7.10.2，再高你就要换系统了。 但是 docker 镜像里只有 7.10.1 版本。

# 安装及测试

# 查询中央仓库
docker search -f is-official=true elasticsearch

# 从中央仓库下载
docker pull elasticsearch:7.11.1

# 或，直接导入已有的镜像文件
# docker load -i elasticsearch-7.11.1.tar

  • 创建并运行 elasticsearch 容器的语法

    docker 运行 elasticsearch 时，有可能会报如下错误：

    max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]

    解决办法是：在 /etc/sysctl.conf 文件最后添加一行：vm.max_map_count=262144 后执行命令 sysctl -p 。

    echo "vm.max_map_count=262144" >> /etc/sysctl.conf

# 容器中的配置文件和数据的存储

Docker ES Server 的配置文件是容器内部：

/usr/share/elasticsearch/config/elasticsearch.yml

该配置文件的内容只有 2 行：

cluster.name: "docker-cluster"
network.host: 0.0.0.0

Docker ES Server 的数据的存储目录是容器内部：

/usr/share/elasticsearch/data

# 配置与挂载

  • 删除曾经已有的同名容器

    docker stop es-9200
docker rm es-9200
    1
    2

    后续可通过 docker run 的 --rm 选项免去『删除』这步操作。

  • 创建本机挂载目录

    注意，要未

    mkdir -p ~/docker/9200/data
touch ~/docker/9200/config/elasticsearch.yml
    1
    2

  • 创建并运行容器

    docker run \
  -d \
  --rm \
  --name es-9200 \
  -p 9200:9200 \
  -p 9300:9300 \
  -v ~/docker/9200/config/elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml \
  -v ~/docker/9200/data:/usr/share/elasticsearch/data \
  -v ~/docker/9200/plugins:/usr/share/elasticsearch/plugins \
  -e "discovery.type=single-node" \
  -e ES_JAVA_OPTS="-Xms256m -Xmx256m" \
  elasticsearch:7.11.1

# 通过 docker-compose 启动

version: '3'

services:
  elasticsearch:
    image: elasticsearch:7.11.1
    container_name: es
    environment:
      - discovery.type: single-node
      - ES_JAVA_OPTS: "-Xms512m -Xmx512m"
    volumes:
      - ./9200/config/elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml
      - ./9200/data:/usr/share/elasticsearch/data
      - ./9200/plugins:/usr/share/elasticsearch/plugins
    ports:
      - "9200:9200"
      - "9300:9300"
    networks:
      - elastic
  kibana:
    image: kibana:7.11.1
    container_name: kib
    ports:
      - "5601:5601"
    depends_on:
      - elasticsearch
    networks:
      - elastic
networks:
  elastic:
    driver: bridge

# Kibana

# 下载 Kibana 镜像

注意，Kibana 的版本需要与 ElasticSearch 保持一致。

docker pull kibana:7.11.1

# 或，直接导入已有的镜像文件
# docker load -i kibana-7.11.1.tar
1
2
3
4

之前要安装 elasticsearch 时，需要创建一个网络，目的是为了让 ElasticSearch 和 Kibana 在同一个网络中：

docker network create es-network
1

运行 Kibana

docker run \
    -d \
    --name kibana \
    --net es-network \
    -p 5601:5601 \
    kibana:7.11.1
